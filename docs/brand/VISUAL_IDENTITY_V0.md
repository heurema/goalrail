# Goalrail Visual Identity v0

> Первая рабочая visual system для Goalrail / Goal on Rails / Punk.
> Не финальный brand book.
> Не жёстко зафиксированная identity.
> Это ship-fast system, которая должна дать визуальную связность и при этом сохранить право на быстрые изменения.

## 1. Purpose

Этот документ фиксирует:
- visual thesis v0
- базовые визуальные принципы
- primary symbols and motifs
- palette direction v0
- typography direction v0
- layout rules v0
- title-card / thumbnail starter patterns
- две tonal configurations, между которыми можно двигаться без пересборки всей системы

## 2. Why this is v0 and not final identity

Visual Identity v0 нужна не для того, чтобы “закрыть бренд навсегда”,
а чтобы:
- перестать каждый раз собирать visuals с нуля
- снизить расползание стиля
- синхронизировать Goalrail, Goal on Rails и Punk
- дать опорную систему для short-form production

Rule:

**v0 должна быть достаточно фиксированной, чтобы давать consistency, и достаточно лёгкой, чтобы быстро перестраиваться.**

## 4. Visual thesis

Current visual thesis:

**controlled delivery in a sharp cyber-industrial frame**

Это значит:
- не glossy SaaS minimalism only
- не chaotic hacker collage
- не gaming neon overload
- не enterprise blue sameness

Goalrail visual identity должна передавать одновременно:
- control
- direction
- boundaries
- motion
- proof
- builder energy

## 5. Core symbolic anchors

### 5.1 Rails
Rails = direction, bounded path, progress under control.

### 5.2 Spikes / porcupine energy
Spikes = sharpness, anti-drift defense, refusal of softness without discipline.

### 5.3 Signals / gates
Signal-light logic = verify, go / no-go, control point, inspection.

### 5.4 Industrial dark surfaces
Dark surfaces = seriousness, focus, technical density without clutter.

### 5.5 Accent energy
Accent color = motion, identity, edge.

## 6. System principles

### Principle 1 — Dark base, controlled accents
Default visuals should sit on dark or near-dark surfaces with restrained accents.

### Principle 2 — One dominant accent, not rainbow branding
Primary accent should remain singular and recognizable.

### Principle 3 — Strong contrast over decorative complexity
Contrast matters more than ornament.

### Principle 4 — Motion cues without visual chaos
The system may imply movement, but must not look uncontrolled.

### Principle 5 — Repetition beats novelty
A small repeatable system is better than constantly inventing new visual language.

## 7. Tonal modes

Visual system v0 supports two tonal modes.

These are not two separate brands.
They are two operating configurations of the same brand system.

### 7.1 Mode A — Serious Business
Use when:
- talking to CTO / Head of Engineering
- pilot explanation
- landing sections with commercial intent
- B2B trust moments

Characteristics:
- darker neutral surfaces
- lower accent pressure
- cleaner spacing
- more restrained compositions
- fewer experimental motifs
- clearer hierarchy

Effect:
- controlled
- credible
- productized
- operator-safe

### 7.2 Mode B — Serious Cyberpunk Fun
Use when:
- build in public clips
- mascot-led content
- sharper thought-leadership content
- social hooks and title cards

Characteristics:
- stronger accent energy
- more visible rails / signal / spike motifs
- more visual edge
- higher contrast punch
- still readable, still controlled

Effect:
- memorable
- sharp
- alive
- non-corporate

### 7.3 Switching rule

Switching modes should happen by changing:
- accent intensity
- motif density
- spacing and composition pressure
- background texture presence

Switching modes should **not** require changing:
- core palette family
- primary symbolic anchors
- type family direction
- identity structure

## 8. Palette direction v0

### 8.1 Base colors
Default base family:
- near-black / graphite
- dark steel / gunmetal
- cool gray
- off-white / light neutral for high-contrast text

### 8.2 Accent family
Primary accent direction:
- electric pink / neon magenta

Why:
- matches Punk mohawk logic
- distinguishes from enterprise-blue SaaS category
- stays memorable across thumbnails and title cards

### 8.3 Secondary accents
Allowed supporting accents:
- warning amber / acid yellow
- cold cyan / signal blue (limited use)

Rule:
secondary accents are supporting signals, not competing brand colors.

### 8.4 Usage rule

Default palette rule:
- 70–80% dark neutrals
- 10–20% light contrast text
- 5–10% primary accent
- secondary accent only when needed for signal separation

## 9. Typography direction v0

### 9.1 General direction
Typography should feel:
- sharp
- modern
- technical but readable
- not retro-futurist parody
- not over-designed

### 9.2 Role split
Recommended role split:
- one display direction for hooks / title cards
- one highly readable text direction for explanations / subtitles / captions in static assets

### 9.3 Hierarchy rule

Headlines:
- short
- strong
- high contrast
- preferably all-caps only when used sparingly

Body copy:
- clean
- tight
- readable
- no decorative distortion

### 9.4 Avoid
Avoid:
- fake “hacker” terminal typography everywhere
- ultra-condensed unreadable display type
- ornamental cyber fonts as default system font
- mixing too many type voices

## 10. Layout system v0

### 10.1 Core layout logic
Default composition should feel like:
- a guided lane
- a controlled signal path
- not a floating collage

### 10.2 Grid preference
Prefer:
- clean vertical or modular grid
- clear zones
- one strong focal point
- one supporting information block

### 10.3 Depth rule
Depth and texture are allowed,
but only if they improve focus.

### 10.4 Empty space rule
Do not fear negative space.
Seriousness often needs room.

## 11. Motif system v0

Allowed recurring motifs:
- rails / track lines
- directional arrows
- signal light / gate markers
- spike silhouettes / sharp contour interruptions
- scanline / subtle technical texture in moderation

Use sparingly:
- glitch
- noise
- distortion
- warning stripes

Rule:
visual edge must serve clarity, not replace it.

## 12. Punk integration rule

Punk can appear in visual system as:
- focal character
- silhouette / partial profile
- accent carrier
- signal interrupter in layout

Punk should not always be present.

Rule:
- product-first assets may use no mascot at all
- mascot-first assets may use Punk strongly
- both must still look like the same system

## 13. Title-card system v0

### 13.1 Purpose
Title cards should make one thought instantly legible.

### 13.2 Structure
Default title-card structure:
- one hook line
- one supporting line or no supporting line
- one focal visual anchor
- one accent treatment

### 13.3 Hook style
Preferred hook style:
- short
- high contrast
- one thought only

Examples of visual-title shapes:
- AI execution got faster. Delivery didn’t.
- Put goals on rails.
- Patch is not proof.
- Don’t let the goal drift.

### 13.4 Background rule
Background should support the headline, not compete with it.

## 14. Thumbnail system v0

### 14.1 Objective
A thumbnail must communicate:
- tension
- one problem
- one strong concept
- recognizability

### 14.2 Default thumbnail types
Type A — text-first
- strong hook
- minimal supporting motif

Type B — Punk + hook
- Punk as focal anchor
- one short line

Type C — system visual
- rails / signals / interface fragment
- product/process idea first

### 14.3 Thumbnail rule
One thumbnail = one promise.
Do not try to summarize the whole video.

## 15. Subtitle-on-video styling guidance v0

This document does not replace subtitle grammar from short-form system,
but it adds styling direction:
- high contrast over dark base
- clean readable text blocks
- no overly decorative subtitle frames
- emphasis only on load-bearing words
- emphasis color should come from accent system, not random highlight colors

## 16. CTA card / end-frame rule

End frames should be simple.

Recommended contents:
- one CTA
- one product or series line
- optional small Goalrail mark / name

Avoid:
- cluttered CTA menu
- too many links
- multiple competing asks

## 17. Flexibility rule

Visual identity v0 must support quick change in:
- motif density
- accent intensity
- Punk presence
- texture amount
- title-card style weight

without forcing rewrite of:
- entire palette
- entire type system
- brand stack
- content grammar

## 18. What this document intentionally leaves open

Still open by design:
- exact hex values
- exact font families
- logo geometry
- icon style system
- final illustration detail level
- exact animation language

These can be fixed later after repeated production use.

## 19. Immediate practical recommendation

For the next production pass:
- use dark graphite base
- use electric pink as primary accent
- keep one readable modern type direction
- define 2–3 repeatable title-card templates
- define 2–3 thumbnail templates
- let Serious Business and Serious Cyberpunk Fun differ by intensity, not by separate brand systems

## 20. Current one-line summary

**Visual Identity v0 should make Goalrail feel like controlled movement through a sharp cyber-industrial frame, with enough consistency to ship and enough flexibility to evolve.**
