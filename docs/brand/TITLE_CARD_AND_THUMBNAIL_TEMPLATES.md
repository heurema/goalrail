# Goalrail Title Card and Thumbnail Templates v0

> Прикладной шаблонный слой после `SHORT_FORM_CONTENT_SYSTEM.md` и `VISUAL_IDENTITY_V0.md`.
> Нужен, чтобы production не собирался с нуля на каждый ролик или карусель.

## 1. Purpose

Этот документ фиксирует:
- repeatable title-card templates
- repeatable thumbnail templates
- когда использовать какой шаблон
- как переключать шаблоны между двумя tonal modes
- какие элементы остаются постоянными
- какие элементы можно менять без расползания системы

## 2. Boundary

Этот документ не заменяет:
- `PUNK_CHARACTER_SYSTEM.md`
- `SHORT_FORM_CONTENT_SYSTEM.md`
- `VISUAL_IDENTITY_V0.md`

Он не определяет brand theory.
Он определяет **production defaults**.

## 3. Current recommendation

Default production set for v0:
- **3 title-card templates**
- **3 thumbnail templates**

Этого достаточно, чтобы:
- давать разнообразие
- не собирать хаос
- быстро запускать серии

## 4. Tonal modes

Все шаблоны должны работать в двух режимах:

### Mode A — Serious Business
Use when:
- pilot / B2B explanation
- CTO-facing value framing
- serious proof / controlled delivery claims

Characteristics:
- lower accent pressure
- cleaner surfaces
- less motif density
- more product/process first

### Mode B — Serious Cyberpunk Fun
Use when:
- build in public
- sharper hook content
- mascot-led content
- social-first concept clips

Characteristics:
- stronger accent energy
- more visible motifs
- more punch in composition
- stronger emotional contrast

Rule:
шаблон остаётся тем же,
меняется его intensity and emphasis.

## 5. Constant elements across all templates

Across all title cards and thumbnails, keep stable:
- dark graphite / near-black base
- one dominant pink accent family
- readable high-contrast type
- one clear focal hierarchy
- one main thought per asset
- rails / spikes / signal motifs only as support, not clutter

## 6. Variable elements across templates

Allowed to vary:
- Punk presence
- motif density
- text size balance
- supporting line presence
- background texture amount
- accent intensity

## 7. Title Card Templates

## Template T1 — Hook Card

### Purpose
Самый универсальный title card.
Нужен для:
- sharp insight
- problem-led clip
- concept-led clip
- короткий narrated build update

### Structure
- one dominant hook line
- optional tiny supporting line
- minimal motif support
- no heavy visual clutter

### Layout logic
- text takes primary focus
- background stays supportive
- motifs stay thin and directional

### Best for
- “Patch is not proof.”
- “Put goals on rails.”
- “AI execution got faster. Delivery didn’t.”

### Serious Business mode
- cleaner background
- smaller accent areas
- no aggressive Punk presence

### Serious Cyberpunk Fun mode
- stronger accent punch
- optional sharper signal or spike interruption
- more tension in spacing

## Template T2 — Problem / Contrast Card

### Purpose
Нужен для сравнений и contrast framing.

### Structure
- left/right or top/bottom contrast
- one tension statement
- optional short clarifier
- visual split or directional tension

### Layout logic
- two opposing states or ideas
- clear “before vs after” / “drift vs rails” / “speed vs control” contrast

### Best for
- “Fast code / weak delivery”
- “Goal drift / goal on rails”
- “Patch / verified result”

### Serious Business mode
- cleaner divide
- more neutral tone
- less decorative energy

### Serious Cyberpunk Fun mode
- stronger split tension
- more dynamic angle or signal contrast
- optional Punk or rail cue

## Template T3 — Build / Proof Card

### Purpose
Нужен для build in public updates и proof-oriented claims.

### Structure
- one clear statement of what changed
- one supporting artifact hint
- optional product/process anchor
- simple CTA-ready composition

### Layout logic
- headline first
- evidence hint second
- clean supporting zone

### Best for
- “We fixed the public narrative layer.”
- “Now the mascot layer is documented.”
- “From goal to verified result.”

### Serious Business mode
- artifact or system cue first
- less mascot energy
- more trust-oriented composition

### Serious Cyberpunk Fun mode
- stronger visual pulse
- optional Punk cameo or sharper accent
- more visible motion cue

## 8. Thumbnail Templates

## Template TH1 — Text-First Thumbnail

### Purpose
Default thumbnail for idea-first content.

### Structure
- one short hook line
- minimal supporting visual
- high readability at small size

### Best for
- abstract idea clips
- concept explainers
- pilot/business narrative clips

### Rule
If the thought itself is the product, use this first.

### Best hooks
- 3–6 words ideal
- 7–9 words max if still instantly readable

## Template TH2 — Punk + Hook Thumbnail

### Purpose
Default thumbnail for mascot-led or stronger social hooks.

### Structure
- Punk as focal anchor
- short hook line
- simple contrast background
- limited motifs only

### Best for
- build in public
- Punk signal content
- sharper anti-drift claims

### Rule
Punk should amplify the idea,
not bury the text.

## Template TH3 — System Visual Thumbnail

### Purpose
Default thumbnail when the process/system itself is the hero.

### Structure
- rails / signal / interface fragment / directional cue as primary visual
- short hook or no hook if the system visual is strong enough
- strong mood / tension

### Best for
- operating layer explanations
- rail metaphor clips
- control / proof / verify concepts

### Rule
Use when the visual metaphor explains the problem better than a face does.

## 9. Template selection rules

Use **T1 / TH1** when:
- the idea is the main payload
- clarity matters more than energy
- the audience is colder / more business-first

Use **T2 / TH2** when:
- Punk should carry extra attention
- content is sharper / more social
- emotional recall matters more

Use **T3 / TH3** when:
- build/proof/system story is the hero
- you want to avoid overusing Punk
- product/process visual metaphor is doing real work

## 10. Text rules for title cards and thumbnails

### 10.1 One thought rule
One asset = one thought.

### 10.2 Word count rule
Preferred headline length:
- 3–6 words ideal
- 7–10 words acceptable
- above that only if still instantly scannable

### 10.3 Density rule
If the card needs a paragraph,
it is not a title card anymore.

### 10.4 Contrast rule
Text must win over decoration.

## 11. Punk usage rules in templates

Punk may appear in:
- TH2 by default
- T2 optionally
- T3 sparingly

Punk should usually not dominate:
- serious B2B trust thumbnails
- proof-first system cards
- product/process explanation assets where face distracts from the model

## 12. Supporting motif rules

Motifs allowed:
- rails
- spike silhouettes
- signal lights
- directional arrows
- subtle scanline / technical texture

Motifs should do one of three jobs only:
- indicate direction
- indicate control point
- add tension edge

If they do not do one of those jobs,
remove them.

## 13. CTA placement rules

### Title cards
Usually no CTA on opening title card.

### End cards
CTA belongs more naturally on end frames.

### Thumbnails
Do not clutter thumbnails with CTA unless absolutely necessary.

## 14. Recommended starter production matrix

### For CTO/business-facing explainers
- Title card: T1 or T2 in Serious Business
- Thumbnail: TH1 or TH3 in Serious Business

### For build in public clips
- Title card: T3 or T1 in Serious Cyberpunk Fun
- Thumbnail: TH2 or TH3 in Serious Cyberpunk Fun

### For Punk-signal clips
- Title card: T1 or T2 in Serious Cyberpunk Fun
- Thumbnail: TH2

## 15. Subtitle compatibility rule

All title-card and thumbnail choices should remain compatible with subtitle readability.

That means:
- do not fill the lower center with unnecessary clutter
- leave room where subtitle blocks may sit in video
- avoid motif placement that competes with subtitle emphasis areas

## 16. Flexibility rule

The starter template set should be treated as:
- stable enough to repeat
- small enough to control
- open enough to evolve

Allowed future expansion:
- T4 if a real new use case appears repeatedly
- TH4 if a real new use case appears repeatedly

Rule:
Do not expand template count just for novelty.

## 17. Immediate recommendation

For the next production cycle, use this default stack:
- **T1** as universal hook card
- **T3** for build and proof updates
- **TH1** for business/process explainers
- **TH2** for Punk-led hooks
- **TH3** for system/metaphor clips

This is the recommended starter set.

## 18. Current one-line summary

**The title-card and thumbnail system should reduce production friction by giving Goalrail a small, repeatable, high-contrast set of templates that work in both Serious Business and Serious Cyberpunk Fun modes.**
