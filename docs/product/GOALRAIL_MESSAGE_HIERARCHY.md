# Goalrail — Message Hierarchy

## 1. What we say first

**Goalrail — единый источник истины для AI-assisted delivery.**

Short variant:

**От бизнес-цели до проверенного изменения в коде.**

## 2. What we say second

Goalrail помогает PM, analyst, tech lead и разработчикам перевести входящую задачу в общий рабочий контракт, а затем провести её через execution, verification и proof.

## 3. What we say third

Продукт не заменяет основных provider tools и не конкурирует с их core capabilities.
Он дополняет их там, где сегодня обычно нет общего server-side control plane:
- intent
- constraints
- contract
- tasks
- runs
- verification
- proof
- business visibility

## 4. What the first public screen must communicate

Не:
- prompt generation
- agent launch
- dashboard control
- AI magic

А:
- **как постановка задачи становится рабочим контрактом**

## 5. Business-first narrative

### Pain
AI ускорил исполнение, но не решил главную проблему: как перевести бизнес-задачу в общий, контролируемый и проверяемый инженерный контур.

### Product answer
Goalrail создаёт единый server-side слой, в котором intent, contract, execution и proof видны и управляемы для бизнеса и инженерии.

### Commercial motion
Не “купить подписку”.
А:
- показать разбор
- прислать свою задачу
- получить пилотный разбор

## 6. What we should never lead with

Не начинать с:
- agents
- models
- memory
- orchestration details
- MCP
- benchmarks
- tooling matrix

Это важные implementation details, но не верхнеуровневый message.

## 7. Message stack by layer

### Hero / one-liner
**От бизнес-цели до проверенного изменения в коде.**

### Subline
Goalrail помогает команде превратить входящую задачу в общий рабочий контракт до начала execution.

### Second layer
Один и тот же контракт для PM, разработчика и агента.

### Third layer
Дальше этот контракт идёт в task shaping, bounded execution, verification и proof.

## 8. Sales-safe language

Хорошо:
- “pilot”
- “разбор вашей задачи”
- “shared working contract”
- “server-side source of truth”
- “supplement layer over provider tools”
- “reviewable and governed delivery”

Осторожно:
- “autonomous engineering”
- “replace your existing tools”
- “we solve everything end to end”
- “deterministic outcome guarantee”
