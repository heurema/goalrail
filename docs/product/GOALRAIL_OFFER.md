# Goalrail Offer

## 1. Purpose

Этот документ фиксирует, **что именно Goalrail продаёт сейчас**, в каком формате, кому это подходит и какой результат клиент получает на выходе.

Он нужен как рабочий источник для:
- qualification conversations
- pilot packaging
- sales deck
- landing rewrite
- outreach и pilot explainer

## 2. Offer in one sentence

**Goalrail сейчас продаётся как free qualification + paid pilot для одной команды, одного репозитория и одного видимого task -> proof workflow.**

Qualification остаётся бесплатным.
Отдельный paid diagnostic package сейчас не активен.

Это не годовая подписка “платформы целиком”.
Это не bespoke consulting под каждого клиента.
Это productized operating layer, который сначала доказывает value на одном bounded кейсе.

## 3. What we sell now

### Entry offer — Free qualification / fit check

Это короткий входной этап, цель которого — быстро понять, подходит ли клиент под стандартный Goalrail deployment.

#### Входит:
- one qualification conversation
- fit check по ICP
- разбор одной реальной команды и одного реального task flow
- проверка sponsor presence
- проверка repo / workflow readiness
- initial deployment profile recommendation
- pilot recommendation: go / no-go

#### Выход:
- fit / no-fit verdict
- pilot candidate definition
- initial deployment profile hypothesis
- список минимальных условий для запуска pilot

#### Не входит:
- длинный аудит всей engineering организации
- redesign процесса под клиента с нуля
- полная техническая интеграция до решения о pilot
- broad consulting engagement

### Diagnostic posture

Отдельный paid diagnostic package пока **не запускается**.

Qualification остаётся бесплатным, чтобы:
- снизить friction в founder-led motion
- быстрее отсеивать no-fit cases
- сохранять первый paid object максимально ясным: **the pilot itself**

## 4. Main offer — Paid pilot

Это первый честный коммерческий продукт Goalrail.

### Default pilot shape
- 1 команда
- 1 repo
- 1 основной кейс
- 2 недели как базовый срок
- managed rollout motion
- remote-first
- lightweight proof required from day one

### Extended pilot shape
- 1 команда
- 1–2 repo
- 2 кейса
- 3–4 недели
- только если нужен более уверенный expansion decision

## 5. What goes into the paid pilot

В pilot входит:
- qualification / fit confirmation
- profile selection
- one deployment configuration
- initial bootstrap
- one onboarding session
- one real pilot case
- one proof output
- one readout with rollout recommendation

### Pilot workflow
Goalrail pilot проходит через один видимый контур:

`incoming task -> working contract -> bounded execution -> verify -> proof`

### Current delivery cell
Ранний pilot по умолчанию продаётся как **founder-led delivery with a founder + DevOps cell**.

Это означает:
- founder ведёт qualification, framing, contract logic и readout
- DevOps участвует в bootstrap, runtime/setup и delivery constraints where needed
- это не staff augmentation и не open-ended embedded team model

## 6. What the client gets

На выходе клиент получает:
- configured Goalrail operating profile
- один реальный contract-driven delivery case
- один proof artifact
- readout of what worked / what blocked
- explicit expansion recommendation

### Minimum proof expectation
Даже в lightweight pilot proof должен отвечать:
- что изменили
- как проверили
- какой verdict
- какие ограничения / риски остались
- стоит ли расширять rollout

## 7. Who this offer is for

Лучший fit сейчас:
- product teams with 5–30 engineers
- есть PM / analyst / tech lead structure
- есть sponsor со стороны CTO / Head of Engineering / VP Engineering
- есть 1–2 repo или реальный task flow для pilot
- уже есть pressure to adopt AI without losing control
- команда готова пройти contract-first workflow

Приемлемые ранние сегменты:
- B2B SaaS product teams
- internal product / platform teams
- engineering studios
- software teams inside larger businesses
- small product teams and startups with one visible repo-level case

## 8. No-fit cases

Goalrail не должен заходить как pilot, если:
- нет sponsor
- нет команды, только один индивидуальный пользователь
- нет реального delivery flow
- нет repo / workflow, на котором можно показать bounded result
- клиент ждёт “полный автопилот”
- клиент ждёт полной замены Jira / Linear / IDE
- security posture делает bounded pilot невозможным без тяжёлого enterprise layer

## 9. Delivery format

### Recommended default
- managed rollout
- remote-first
- bounded pilot scope
- limited configuration over a fixed operating core
- founder-led pilot with founder + DevOps delivery cell

### Optional later
- guided rollout
- onsite support only when really needed
- wider rollout after successful pilot

Goalrail не должен заходить как:
- full organization redesign
- replacement of the whole toolchain
- long custom audit before first result
- open-ended consulting with no bounded proof target

## 10. Commercial posture

### Pricing posture
- **qualification stays free**
- **paid diagnostic is not active yet**
- **paid pilot is the main commercial object**

### Why this structure
- qualification быстро убирает no-fit cases
- основной value создаётся внутри pilot execution
- paid pilot лучше валидирует seriousness и rollout intent

### What we do not sell first
- annual platform subscription as the first object
- full enterprise rollout before pilot proof
- broad platform implementation across many teams at once

## 11. Current pricing rule

Текущая pricing-логика должна быть:
- **USD-anchored first**
- **public anchor: Managed pilot — from $5,000**
- qualification remains free
- design partner is a discount mode for the same pilot offer, only when strategically justified
- design partner is not used to rescue weak-fit accounts
- основной коммерческий якорь — managed pilot, а не subscription

Детальная ценовая рамка вынесена в:
- `docs/product/GOALRAIL_PRICING_MODEL.md`

## 12. Expansion path

После pilot есть только три честных исхода:
- **expand** — добавить второй repo, второй кейс или вторую команду
- **stabilize and retry** — подкрутить profile / policy / templates и повторить bounded rollout
- **stop** — зафиксировать no-fit без искусственного продолжения engagement

### Typical expansion directions
- second repo
- second use case
- second team
- stronger review / policy profile
- more stable deployment playbook

## 13. What this offer is not

Goalrail offer сейчас — это не:
- self-serve SaaS page
- “ещё один AI tool”
- prompt-generation service
- generic agent framework deployment
- bespoke consulting per company

Это:
- productized operating layer
- contract-centered pilot
- bounded execution + proof service contour
- controlled path from incoming task to inspectable result
