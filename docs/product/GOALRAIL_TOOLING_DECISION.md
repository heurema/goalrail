# GOALRAIL — Tooling Decision (Stitch vs Figma)

## Decision

At the current stage:

- **Stitch** is the primary tool for exploring the public 2-scene concept
- **Figma** is the tool for fixing and polishing the chosen direction

## Rationale

The current problem is still partly conceptual:

- what exactly is the product object shown on screen
- how scene 1 and scene 2 connect
- how to avoid prompt-tool / dashboard / website drift

Stitch is currently better for fast multi-scene generation and variation.

Figma becomes more useful **after** a direction is chosen:

- copy lock
- layout lock
- hierarchy
- spacing
- design cleanup
- prototype / handoff

## Practical workflow

1. Generate alternatives in Stitch
2. Choose one baseline
3. Move that baseline into Figma
4. Refine manually / with AI in Figma
5. Build prototype / final page from the Figma baseline
