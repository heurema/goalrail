# Support

## Where to ask for help

Use the repository issue forms for:

- bug reports;
- feature requests;
- bounded questions about the repository, contribution flow, or current scope.

## Before opening a question

Please check these first:

- `README.md`
- `docs/INDEX.md`
- `CONTRIBUTING.md`
- `SECURITY.md` (for security issues)
- relevant product / ops docs linked from `docs/INDEX.md`

## When not to use public issues

Do **not** use public issues for:

- security vulnerabilities;
- private customer or pilot details;
- sensitive credentials, tokens, logs, or datasets.

Use `hello@goalrail.dev` for security and other sensitive contact.

## Response expectations

This is an open source project with finite maintainer capacity. A response is
not guaranteed on a particular timeline, but well-scoped reports and well-
documented questions are much easier to address.
