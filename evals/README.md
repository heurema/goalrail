# Evals

Reserved for future verification cases, specs, and suites.

Current rule:
- `evals/` documents planned evaluation surfaces and test intent
- it does not imply a runnable Goalrail harness exists yet

## Layout

- `evals/specs/` — evaluation notes and planned policies
- `evals/cases/` — future case fixtures
- `evals/suites/` — future suite definitions
- `evals/_templates/` and `evals/_schema/` — future helpers
