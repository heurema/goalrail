# Evals

Reserved for future verification cases, specs, and suites.

Current rule:
- `.goalrail/evals/` documents planned evaluation surfaces and test intent
- it does not imply a runnable Goalrail harness exists yet

## Layout

- `.goalrail/evals/specs/` — evaluation notes and planned policies
- `.goalrail/evals/cases/` — future case fixtures
- `.goalrail/evals/suites/` — future suite definitions
- `.goalrail/evals/_templates/` and `.goalrail/evals/_schema/` — future helpers
