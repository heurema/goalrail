# Goalrail illustrated story style

Scope: visual style for LinkedIn illustrations and profile assets in the Goalrail origin series.

## Purpose

Each image should work as a standalone visual chapter. The reader should understand the core situation before reading the post.

The illustrations should continue one story, not look like unrelated marketing graphics.

## Formats

### Feed posts

- Primary format: vertical 4:5.
- Standard export: 1080 x 1350 px.
- Reference asset: `.punk/publishing/assets/linkedin/2026-04-24-goal-on-rails-origin.png`.

### LinkedIn profile banner

- Standard export: 1584 x 396 px.
- Keep the lower-left area visually safe because the profile photo overlaps the banner.
- Important content should live center and right.
- Reference asset: `.punk/publishing/assets/linkedin/2026-04-24-goalrail-profile-banner.png`.

### LinkedIn profile avatar

- Square export.
- Standard working export: 800 x 800 px.
- Keep the face centered and readable in a circular crop.
- Reference asset: `.punk/publishing/assets/linkedin/2026-04-24-goalrail-profile-avatar.png`.

## Visual direction

- Premium editorial B2B technology illustration.
- Business-facing, calm, modern, confident.
- Not a comic strip.
- Not a meme.
- Not cyberpunk.
- Not generic SaaS stock art.

## Palette

Base the palette on the real Goalrail demo UI.

- deep charcoal
- dark graphite
- warm dark brown
- muted gray
- warm off-white only as a small contrast surface
- amber or gold as the single main accent

Avoid bright saturated colors.
Avoid blue-purple generic AI gradients unless the actual product style changes.

## Recurring motifs

Use these motifs across the series:

- rails or guided tracks
- goal card or goal token
- contract/workflow checkpoints
- dark Goalrail-like admin screen
- amber active path
- human in the loop
- notebook or working notes
- proof, checks, verification, approval

The rail should mean bounded execution from goal to verified change.

## Product surface

When showing a screen, it should be grounded in the actual Goalrail admin style:

- dark UI
- contract or workspace surface
- progress stages
- panel borders
- amber active state
- proof or readiness indicators
- minimal typography

Do not copy sensitive implementation details literally.
Use simplified UI inspired by the demo.

## Human subject

The founder can appear in two modes:

1. Rear three-quarter observer in the feed illustrations.
2. Direct portrait for the LinkedIn avatar.

In feed illustrations, the face should not dominate. The person is a reviewer/operator inside the story, not a hero poster.

## Series arc for images

Episode 1: AI inside IDE, trust still lives in diff, rails begin to appear.
Episode 2: diff review becomes a bottleneck, output grows faster than attention.
Episode 3: goal intake, business intent becomes a contract.
Episode 4: clarification and ambiguity inspector, scope gets bounded.
Episode 5: execution under constraints, agent work stays inside rails.
Episode 6: verification and proof feed, evidence is produced.
Episode 7: approval and decision, team accepts only what is proven.
Episode 8: live team pilot, one repo and one workflow shape the product defaults.

## Composition rules

- One clear scene, one dominant idea.
- The person can appear as an observer/reviewer, not as a superhero.
- The monitor should show real work, not decorative nonsense.
- Use rail path from left to right or foreground to screen to show progression.
- Keep labels minimal because AI image text can be unreliable.
- Image must remain readable at feed size.

## Negative prompts

Avoid:

- comic panels
- speech bubbles
- cartoon faces
- exaggerated robots
- random glowing code rain
- fake futuristic city
- heavy readable UI text
- too many icons
- overly cheerful startup illustration
- stock-photo realism

## Reference prompt core

Create a premium editorial illustration for a LinkedIn post series about Goalrail / Goal on Rails. Use the real Goalrail demo admin as the primary UI style reference. Match the dark graphite-to-brown interface theme, warm amber accents, subtle panel borders, minimal typography feel, and overall Goalrail visual language. Show a thoughtful professional at a desk looking at a large screen with a simplified Goalrail-style contract/workflow surface. A rail-like path moves from a goal token through checkpoint nodes toward the monitor. The image should communicate work moving from goal toward verified change along rails into a real system. Vertical 4:5. Business-facing, calm, modern, not comic, not cyberpunk. Use deep charcoal, warm brown, muted gray, and amber accent.
